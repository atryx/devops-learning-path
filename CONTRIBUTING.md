# Contributing to DevOps Learning Path

Thanks for helping improve this roadmap!

## How to Contribute

1. **Fork** the repo
2. **Create a branch** (`git checkout -b improve/section-name`)
3. **Edit** the README — add resources, fix outdated info, or suggest new sections
4. **Submit a PR** with a clear description

## Guidelines

- Keep recommendations **practical** — tools people actually use in production
- Link to **free resources first**, paid ones second
- Include real-world context for tool choices
- No affiliate links

## What We're Looking For

- Updated tool recommendations for 2026
- Hands-on project ideas at each level
- Missing topics or skills
- Better learning resource links
